What are Props?

Props (short for "properties") are how you pass data from parent components to child components in React. Think of props as function arguments - they allow you to customize how a component behaves and what it displays.

üí° Key Concept: Props are read-only! A component should never modify its own props. This keeps your application predictable and easier to debug.

Basic Props Usage

Let's start with a simple example of passing props to a component:

// Child component that receives props function Greeting(props) { return <h1>Hello, {props.name}!</h1>; } // Parent component that passes props function App() { return ( <div> <Greeting name="Alice" /> <Greeting name="Bob" /> <Greeting name="Charlie" /> </div> ); }

Destructuring Props

Instead of using props.name, you can destructure props to make your code cleaner:

Without Destructuring

function UserCard(props) { return ( <div> <h2>{props.name}</h2> <p>Age: {props.age}</p> <p>Email: {props.email}</p> </div> ); }

With Destructuring

function UserCard({ name, age, email }) { return ( <div> <h2>{name}</h2> <p>Age: {age}</p> <p>Email: {email}</p> </div> ); }

Props Validation with PropTypes

You can validate props to catch bugs early in development:

import PropTypes from 'prop-types'; function ProductCard({ name, price, category, inStock, onAddToCart }) { return ( <div className="product-card"> <h3>{name}</h3> <p>Price: ${price}</p> <p>Category: {category}</p> {inStock ? ( <button onClick={onAddToCart}>Add to Cart</button> ) : ( <p>Out of Stock</p> )} </div> ); } // Prop validation ProductCard.propTypes = { name: PropTypes.string.isRequired, price: PropTypes.number.isRequired, category: PropTypes.string, inStock: PropTypes.bool, onAddToCart: PropTypes.func }; // Default props ProductCard.defaultProps = { category: 'General', inStock: true, onAddToCart: () => {} };
üí° Pro Tip: PropTypes are only checked in development mode and help you catch bugs early. They're removed in production builds for performance.

The Children Prop

React provides a special prop called children that contains the content between component tags:

// Wrapper component that uses children function Card({ title, children }) { return ( <div className="card"> <div className="card-header"> <h3>{title}</h3> </div> <div className="card-body"> {children} </div> </div> ); } // Using the Card component function App() { return ( <div> <Card title="Welcome"> <p>This is the card content!</p> <button>Learn More</button> </Card> <Card title="Features"> <ul> <li>Easy to use</li> <li>Fast performance</li> <li>Great support</li> </ul> </Card> </div> ); }

Props Best Practices

  • Keep props simple: Pass only the data that the component needs
  • Use descriptive names: isLoading is better than loading
  • Avoid prop drilling: Don't pass props through many levels unnecessarily
  • Use PropTypes: Validate your props to catch errors early
  • Provide defaults: Use default parameters for optional props
  • Keep components pure: Same props should always produce the same output
üéØ Remember: Props flow down from parent to child components. If you need to send data up from child to parent, you'll use callback functions (which we'll cover in the next lesson about state and events).
