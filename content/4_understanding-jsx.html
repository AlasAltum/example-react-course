What JSX Really Is

JSX is syntactic sugar for JavaScript. When you write JSX, it gets transformed into regular JavaScript function calls. Understanding this transformation helps you write better React code.

JSX (What You Write)

const element = ( <h1 className="greeting"> Hello, world! </h1> );

JavaScript (What It Becomes)

const element = React.createElement( 'h1', {className: 'greeting'}, 'Hello, world!' );

Embedding Expressions in JSX

You can embed any JavaScript expression in JSX by wrapping it in curly braces {}:

function WelcomeMessage() { const user = { firstName: 'John', lastName: 'Doe', age: 25 }; const formatName = (user) => { return user.firstName + ' ' + user.lastName; }; return ( <div> <h1>Welcome, {formatName(user)}!</h1> <p>You are {user.age} years old.</p> <p>Next year you'll be {user.age + 1}.</p> <p>Random number: {Math.floor(Math.random() * 100)}</p> </div> ); }

Conditional Rendering

You can conditionally render elements using JavaScript operators:

1. Using the Ternary Operator

function UserGreeting({ isLoggedIn, username }) { return ( <div> {isLoggedIn ? ( <h1>Welcome back, {username}!</h1> ) : ( <h1>Please sign in.</h1> )} </div> ); }

2. Using the Logical AND Operator

function Notifications({ messages }) { return ( <div> <h1>Hello!</h1> {messages.length > 0 && ( <h2>You have {messages.length} unread messages.</h2> )} </div> ); }

Handling Events

React events are similar to HTML events but use camelCase naming:

function InteractiveComponent() { const handleClick = () => { alert('Button clicked!'); }; const handleInputChange = (event) => { console.log('Input value:', event.target.value); }; const handleSubmit = (event) => { event.preventDefault(); // Prevent form submission console.log('Form submitted!'); }; return ( <div> <button onClick={handleClick}> Click me </button> <input type="text" onChange={handleInputChange} placeholder="Type something..." /> <form onSubmit={handleSubmit}> <input type="text" placeholder="Enter data" /> <button type="submit">Submit</button> </form> </div> ); }

JSX Best Practices

  • Keep JSX readable: Break complex JSX into smaller components
  • Use meaningful variable names: isLoading instead of loading
  • Extract complex logic: Move complex expressions to variables or functions
  • Use fragments: Use <></> to avoid unnecessary div wrappers
  • Format consistently: Use Prettier or similar tools for consistent formatting
