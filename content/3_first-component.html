What is a React Component?

A React component is like a JavaScript function that returns HTML. It's a reusable piece of code that describes what should appear on the screen. Think of components as custom HTML elements that you can create and use throughout your application.

üí° Key Concept: Components let you split the UI into independent, reusable pieces. Each component can have its own logic and styling, making your code more organized and maintainable.

Function Components vs Class Components

There are two ways to define components in React:

Class Component (Old Way)

class Welcome extends React.Component { render() { return <h1>Hello, World!</h1>; } }

Function Component (Modern Way)

function Welcome() { return <h1>Hello, World!</h1>; }

We'll focus on function components as they're simpler and the modern standard in React development.

Creating Your First Component

Let's create a simple greeting component. Open your src/App.js file and replace its contents with:

import React from 'react'; import './App.css'; // Our first custom component function Greeting() { return ( <div> <h1>Hello, React Developer!</h1> <p>Welcome to your first custom component.</p> </div> ); } // Main App component function App() { return ( <div className="App"> <Greeting /> </div> ); } export default App;
‚úÖ Result: Save the file and check your browser. You should see your custom greeting message displayed on the page!

Understanding JSX

JSX (JavaScript XML) is a syntax extension for JavaScript that looks like HTML but has the power of JavaScript. It's what makes React components so intuitive to write.

JSX Rules to Remember

  • Single Parent Element: JSX must return a single parent element
  • className instead of class: Use className for CSS classes
  • Self-closing tags: Tags like <img> must be self-closed: <img />
  • JavaScript expressions: Use curly braces {} to embed JavaScript
// ‚ùå Wrong - Multiple parent elements function BadComponent() { return ( <h1>Title</h1> <p>Paragraph</p> ); } // ‚úÖ Correct - Single parent element function GoodComponent() { return ( <div> <h1>Title</h1> <p>Paragraph</p> </div> ); } // ‚úÖ Also correct - Using React Fragment function AlsoGoodComponent() { return ( <> <h1>Title</h1> <p>Paragraph</p> </> ); }

Adding JavaScript to JSX

You can embed any JavaScript expression inside JSX using curly braces:

function PersonalGreeting() { const name = "Sarah"; const currentTime = new Date().toLocaleTimeString(); return ( <div> <h1>Hello, {name}!</h1> <p>Current time: {currentTime}</p> <p>2 + 2 = {2 + 2}</p> </div> ); }

Best Practices for Components

  • Component Names: Always start with a capital letter (PascalCase)
  • One Component Per File: Keep components in separate files for better organization
  • Descriptive Names: Use names that clearly describe what the component does
  • Keep It Simple: Each component should have a single responsibility
üéØ Pro Tip: If your component is getting too large or doing too many things, it's probably time to break it down into smaller components. This makes your code easier to understand, test, and maintain.

Common Beginner Mistakes

  • Forgetting to capitalize component names: greeting vs Greeting
  • Using class instead of className: This will cause warnings
  • Forgetting to close self-closing tags: <img> should be <img />
  • Not wrapping multiple elements: Always have a single parent element
